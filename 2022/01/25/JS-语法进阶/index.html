本节主要关注JS基础语法中进阶的部分，包括数据类型转换、模板字面量、Symbol数据类型、for-in/of与with语句

1 数据类型

ES有6种简单数据类型：Undefined、Null、Boolean、Number、String、Symbol。还有一种复杂数据类型Object（无序名值对的集合）。

Underfine：

Undefined类型只有一个值，就是特殊值undefined。当使用var或let声明了变量但没有初始化时，就相当于赋值undefined。

Null：

null同样只有一个值：null。

null值表示一个空对象指针，因此在定义将来要保存对象值的变量时，建议使用null来初始化。

Boolean：

有两个字面值：true、false。这两个布尔值不同于数值，true不等于1，false不等于0。

1.1 Number类型

用来表示各种进制的整数和浮点数。

  • 八进制第一个数字必须是0.

  • 十六进制开头为0x，然后是十六进制的数字。

  • 科学计数法：一个数值（整数/浮点数）+e+10的幂数

  • Infinity表示无限

  • NaN表示不是数值，isNaN()判断是不是数字，false表示是数字

类型转换：

  • 我们可以使用 Number()函数将类型转换成数字类型，如果参数无法被转换为数字，则返回 NaN。具体规则如下：

    参数类型 结果
    Undefined NaN
    Null +0
    Boolean 如果参数是 true，返回 1。参数为 false，返回 +0
    Number 返回与之相等的值
    String 如果通过 Number 转换函数传入一个字符串，它会试图将其转换成一个整数或浮点数，而且会忽略所有前导的 0，如果有一个字符不是数字，结果都会返回 NaN

  • parseInt 只解析整数，parseFloat 则可以解析整数和浮点数，如果字符串前缀是 “0x” 或者”0X”，parseInt 将其解释为十六进制数，parseInt 和 parseFloat 都会跳过任意数量的前导空格，尽可能解析更多数值字符，并忽略后面的内容。如果第一个非空格字符是非法的数字直接量，将最终返回 NaN：

浮点数：

ECMAScript 中的 Number 类型使用 IEEE754 标准来表示整数和浮点数值。所谓 IEEE754 标准，全称 IEEE 二进制浮点数算术标准，这个标准定义了表示浮点数的格式等内容。

在 IEEE754 中，规定了四种表示浮点数值的方式：单精确度（32位）、双精确度（64位）、延伸单精确度、与延伸双精确度。像 ECMAScript 采用的就是双精确度，也就是说，会用 64 位来储存一个浮点数。

因为存储浮点值使用的内存空间是存储整数值的两倍，所以ECMAScript 总是想方设法把值转换为整数。在小数点后面没有数字的情况下，数值就会变成整数。

浮点值的精确度最高可达17 位小数，但在算术计算中远不如整数精确。例如，0.1 加0.2 得到的不是0.3，而是0.300 000 000 000 000 04。由于这种微小的舍入错误，导致很难测试特定的浮点值。

1.2 String类型

类型转换：

  • toString()方法：是Object、Array、Number、String、Function的原型方法。这个方法返回当前值的字符串等价物。例如：

    1. 数组的 toString 方法将每个数组元素转换成一个字符串，并在元素之间添加逗号后合并成结果字符串。
    2. 函数的 toString 方法返回源代码字符串。
    3. 日期的 toString 方法返回一个可读的日期和时间字符串。
    4. RegExp 的 toString 方法返回一个表示正则表达式直接量的字符串

  • Object.prototype.valueOf()：默认的 valueOf 方法返回这个对象本身，数组、函数、正则简单的继承了这个默认方法，也会返回对象本身。日期是一个例外，它会返回它的一个内容表示: 1970 年 1 月 1 日以来的毫秒数。

  • JSON.stringify()：方法将一个 JavaScript 对象或值转换为 JSON 字符串，如果指定了一个 replacer 函数，则可以选择性地替换值，或者指定的 replacer 是数组，则可选择性地仅包含数组指定的属性。

模板字符串：

ES6中新增的字符串定义方式，以``定义，可以跨行定义字符串。

最常用的特性就是字符串插值，可以在一个连续定义中插入一个或多个值。

模板字面量也支持标签函数，标签函数会接收到参数一次是原始字符串数组和对每个表达式求值的结果，这个函数的返回值是对模板求值得到的字符串

因为表达式参数数量是可变的，通常使用扩展操作符把他们收集到一个数组中。

1.3 Symbol类型

Symbol（符号）是ECMAScript 6 新增的数据类型。符号是原始值，且符号实例是唯一、不可变的。符号的用途是确保对象属性使用唯一标识符，不会发生属性冲突的危险。Symbol特性如下：

  1. Symbol 值通过 Symbol 函数生成，使用 typeof，结果为 “symbol”

  2. Symbol 函数前不能使用 new 命令，否则会报错。这是因为生成的 Symbol 是一个原始类型的值，不是对象。因此instanceof 的结果为 false

  3. Symbol 函数可以接受一个字符串作为参数，表示对 Symbol 实例的描述，主要是为了在控制台显示，或者转为字符串时，比较容易区分。

  4. 如果 Symbol 的参数是一个对象，就会调用该对象的 toString 方法，将其转为字符串，然后才生成一个 Symbol 值。

  5. Symbol 函数的参数只是表示对当前 Symbol 值的描述，相同参数的 Symbol 函数的返回值是不相等的。

  6. Symbol 值不能与其他类型的值进行运算，会报错。

  7. Symbol 值可以显式转为字符串。

  8. Symbol 值可以作为标识符，用于对象的属性名，可以保证不会出现同名的属性。

  9. Symbol 作为属性名，该属性不会出现在 for…in、for…of 循环中，也不会被 Object.keys()、Object.getOwnPropertyNames()、JSON.stringify() 返回。但是，它也不是私有属性，有一个 Object.getOwnPropertySymbols 方法，可以获取指定对象的所有 Symbol 属性名。

  10. 如果我们希望使用同一个 Symbol 值，可以使用 Symbol.for。它接受一个字符串作为参数，然后搜索有没有以该参数作为名称的 Symbol 值。如果有，就返回这个 Symbol 值，否则就新建并返回一个以该字符串为名称的 Symbol 值。

  11. Symbol.keyFor 方法返回一个已登记的 Symbol 类型值的 key。
