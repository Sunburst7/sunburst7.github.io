2 Proxy and Reflect

在上一节中，我们了解了 Vue 3 如何跟踪effect以在需要时重新运行它们。 但是，我们仍然需要手动调用 track 和 trigger。 在本课中，我们将学习如何使用 Reflect 和 Proxy 自动调用它们。

2.1 监听Get and Set

我们需要一种方法来Hook（或监听）我们的反应对象上的 get 和 set 方法。

  • GET property：调用track()收集依赖(effect)

  • SET property：调用trigger()重新触发所有已经收集的依赖(effect)

在 Vue 2 中，我们使用 ES5 Object.defineProperty 来做到这一点。在 Vue 3 中我们使用 ES6 Reflect 和 Proxy 拦截 GET 和 SET 调用。

3.2 ref

思考一个问题：在计算total时我们应该根据salePrice计算而不是product.price。

但如果这么做，我们构造的响应式系统就会失效，当修改product.price时，由于其依赖集合中没有total的计算代码，因此不会重新渲染total。而且salePrice是非响应式的变量，更不会修改。

介绍一下ref API

  • ref：接受一个内部值并返回一个响应式且可变的 ref 对象。ref 对象仅有一个 .value property，指向该内部值。

  • toRefs

    将响应式对象转换为普通对象，其中结果对象的每个 property 都是指向原始对象相应 property 的 ref。

    当从组合式函数返回响应式对象时，toRefs 非常有用，这样消费组件就可以在不丢失响应性的情况下对返回的对象进行解构/展开：
